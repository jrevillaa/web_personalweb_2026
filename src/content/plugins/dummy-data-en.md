## General Description

**Dummy Data Generator** accelerates QA and load testing: creates a representative environment without manual effort.

### Best Practices

1. **Reproducibility**
   * Configurable seeds for repeatable datasets.
2. **Security**
   * Generates fictitious data, no real PII.
3. **Scalability**
   * Jobs via cron or CLI to create huge volumes without timeouts.
4. **Cleanup**
   * Rollback/cleanup options to iterate fast.
